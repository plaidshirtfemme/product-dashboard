# Product Analyst — роль в модельной команде

## Место в команде

Product Analyst (PA) закрывает **разрыв между Research и BA**:

| Роль | Фокус | Что не покрывает |
|---|---|---|
| Researcher / UX | Качественные инсайты, интервью, JTBD | Цифры, статистика, SQL |
| **Product Analyst** | **Данные, A/B тесты, метрики, BI** | **Качественный контекст, требования** |
| BA / SA | Требования, процессы, архитектура | Аналитика данных |

PA не заменяет Researcher и не пишет user stories — он переводит данные в решения.

---

## Что делает PA в нашем пайплайне

### 1. Analysis tab (основной дом PA)
- Воронки конверсии (Activation Funnel) — строит и интерпретирует
- A/B тесты — статистическая значимость, sample size, итоги → Kill/Scale решение
- Cohort retention — разбивка по когортам, кривые удержания
- SQL-запросы к ClickHouse / product DB

### 2. OKR / Overview tab
- Отслеживает Key Results в реальном времени: текущее vs target vs baseline
- Предупреждает PM о статусе at_risk / off_track до конца квартала
- Связывает OKR с метриками продукта (что конкретно двигает KR)

### 3. Growth tab
- Ведёт Experiment Log: варианты, результаты, решения Scale/Kill
- Рассчитывает win rate и статистическую значимость
- Формулирует следующие гипотезы на основе данных

### 4. Backlog
- DoD-чеклист: инструментация добавлена? (колонка 📊)
- Связывает каждую Story с метрикой, которую она должна сдвинуть

### 5. Monitoring
- Настраивает дашборды и алерты (ClickHouse + Grafana / Redash)
- Анализирует связь Support-тикетов с багами — приоритизирует через данные

## Как PA взаимодействует с другими ролями

```
Researcher → [качественные инсайты] → PA → [количественное подтверждение] → PM
PM          → [гипотеза]             → PA → [дизайн эксперимента]           → Dev
Dev         → [релиз фичи]           → PA → [измерение результата]           → Growth
```

- **С PM**: ежедневный доступ к метрикам, еженедельный OKR review
- **С Researcher**: валидирует качественные инсайты через data (n=?)
- **С BA**: BA пишет требования → PA пишет acceptance criteria через метрики
- **С Growth**: совместное ведение Experiment Log, Kill/Scale решения

## Что сейчас отсутствует в дашборде для PA

| Что нужно | Где добавить | Статус |
|---|---|---|
| Реальная воронка активации | Analysis / Growth | ⚠️ мок, нет product analytics |
| Статистическая значимость экспериментов | Growth → Experiment Log | ❌ нет данных |
| Cohort retention chart | Analysis tab | ❌ нет |
| SQL query explorer | Analysis tab | ❌ нет |
| Метрики per-feature (impact) | Backlog → issue row | ❌ нет |
| DoD check: инструментация | Backlog колонка 📊 | ✅ добавлено (mock) |

## Профиль вакансии (Muse Group / Product Analyst)

На основе анализа открытых позиций ключевые компетенции:

**Hard skills**: SQL (ClickHouse), Python/R, A/B testing, статистика, ETL/dbt, BI-инструменты  
**Soft skills**: превращать данные в решения, презентовать PM и стейкхолдерам  
**Домен**: продуктовая аналитика (не финансовая, не маркетинговая)

Типичный вход в команду: после 2–3 лет в аналитике данных / BI, с опытом в product-компании.
