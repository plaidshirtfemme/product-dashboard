# Аналитика посещений + тепловые карты (DASH-125)

> Исследование 13.07.2026. Как мерить вовлечённость рекрутеров на двух поверхностях:
> портфолио на Framer и дашборд на Reflex Cloud. Что видно, чем и как подключить.

## TL;DR — рекомендация

- **Тепловые карты + записи сессий: Microsoft Clarity** — бесплатно, без лимитов, ставится одним скриптом на ОБЕ поверхности. Показывает: куда кликают, докуда доскроллили, записи сессий. Плюс даёт базовую аналитику (просмотры, популярные страницы) — то есть **один инструмент закрывает и heatmaps, и базовые метрики**.
- **Базовая аналитика портфолио:** встроенная Framer Analytics (бесплатно, уже есть) — можно оставить как дополнение.
- **Оговорка приватности:** Clarity бесплатен, потому что Microsoft использует данные для обучения своих AI. Для публичного трафика портфолио (рекрутеры листают) — чувствительность низкая. Если некомфортно — альтернативы ниже.

## Поверхность 1 — Портфолио на Framer

- **Встроенная Framer Analytics** — на ВСЕХ планах, включая Free. Базовое: просмотры, топ-страницы, источники. Хранение: Free/Basic ~30 дней, Pro ~90. Advanced Analytics — триал 30 дней, потом биллинг по событиям (25k событий/мес бесплатно). → базовые визиты видны бесплатно.
- **Heatmaps — НЕ встроены** в Framer. Нужен сторонний инструмент через custom-скрипт.
- **Google Analytics** — есть отдельное поле «Measurement ID» в Site Settings (просто вставить ID).
- **Clarity/Hotjar/Plausible** — через Framer → Site Settings → custom code (вставка `<script>` в head).

## Поверхность 2 — Дашборд на Reflex

Любой JS-скрипт (Clarity, GA, Plausible) ставится через `head_components` приложения:

```python
# dash_app/dash_app.py
import reflex as rx
CLARITY = rx.script("""(function(c,l,a,r,i,t,y){ ...сниппет Clarity... })
(window, document, "clarity", "script", "ВАШ_ID");""")
app = rx.App(head_components=[CLARITY])
```

- `rx.App(head_components=[rx.script(...)])` вставляет скрипт в `<head>` всех страниц.
- После правки — **передеплой** (`python scripts/deploy.py`).
- Важно: это **JS-инструменты**, поэтому пингер им не мешает (да его и нет — см. DASH-103). Статистика чистая.

## Сравнение инструментов (heatmaps + записи)

| Инструмент | Цена | Что даёт | Хранение | Приватность |
|---|---|---|---|---|
| **Microsoft Clarity** ⭐ | **Бесплатно, без лимитов** | heatmaps, записи сессий, база-аналитика | ~90 дней | MS обучает AI на данных |
| **Hotjar** | Free (35 сессий/день) | + опросы, фидбэк, воронки | 365 дней | privacy-first |
| **PostHog** | Free (5000 записей/мес) | + продуктовая аналитика, воронки; self-host | 30 дней | полный контроль при self-host |

**Аналитика просмотров (если нужна отдельно от heatmaps):**
- **Plausible / Fathom** — privacy-first, лёгкие, без cookie-баннера, ~$9/мес.
- **Google Analytics (GA4)** — бесплатно, мощно, но тяжелее + нужен cookie-consent.
- **PostHog** — аналитика + heatmaps в одном, free tier.

## Итог по выбору

- **Хочешь минимум усилий + бесплатно + heatmaps:** Microsoft Clarity на обе поверхности (Framer custom script + Reflex head_components). Один инструмент, $0, покрывает «куда кликают / докуда скроллят / записи + базовые метрики».
- **Не хочешь отдавать данные Microsoft:** heatmaps → PostHog (free, self-host для полного контроля) или Hotjar (privacy-first); аналитика → Plausible.
- **Тайминг:** post-publish / 2-я итерация — аналитику можно добавить после запуска (вс 19), она не блокирует публикацию.

## Шаги для внедрения (когда дойдём)
1. Завести аккаунт в выбранном инструменте (Clarity — вход через Microsoft/Google), получить ID/сниппет.
2. **Framer:** Site Settings → custom code → вставить сниппет в head.
3. **Reflex:** добавить `rx.script(сниппет)` в `head_components` в dash_app.py → `python scripts/deploy.py`.
4. Проверить, что данные капают (Clarity dashboard) — записи появляются через ~пару часов.
